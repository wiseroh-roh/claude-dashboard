# 좌측 사이드바 네비게이션 + 메모리 UI 개선 — 설계

**날짜:** 2026-06-14
**대상:** `claude-dashboard` 프론트엔드 (`public/`)
**범위:** UI 전용. 서버(`src/`)·API·데이터 소스는 변경 없음.

## 배경

현재 대시보드는 세로 1단 구조다: `헤더(app-bar) → KPI 바 → 가로 탭 → 본문(패널)`.
탭이 본문 위에 가로로 놓여 있어, 화면이 넓어질수록 좌측 공간이 비고 메뉴와 내용의
시선 이동이 위아래로 길어진다. 또 메모리 탭은 항목을 단순 `ul` 목록으로 나열해
유형 구분(사용자/피드백/프로젝트/참고)이 시각적으로 약하다.

## 목표

1. 탭 메뉴를 **좌측 사이드바**로 옮기고, 현재 선택된 메뉴를 강조한다.
2. 메모리 탭을 **유형별 색상 카드 그리드**로 바꿔 가독성과 미관을 높인다.

비목표(YAGNI): 다국어 토글, 사이드바 접기 토글 버튼, 메모리 검색/필터, 신규 API.

## 1. 레이아웃 — 좌측 사이드바 (A안)

세로 구조를 2단 CSS 그리드(`사이드바 | 본문`)로 재구성한다.

```
┌──────────┬───────────────────────────────┐
│ ✦ Claude │            [CC v… chip] (우측) │  ← 본문 상단 바
│ Dashboard│  ┌────┬────┬────┬────┬────┐    │
│          │  │KPI │KPI │KPI │... │    │    │  ← KPI 바 (기존 7개)
│ ▸ 세션    │  └────┴────┴────┴────┴────┘    │
│   MCP    │                               │
│   스킬    │     [활성 패널 내용]            │
│   메모리   │                               │
│   설치    │                               │
│          │                               │
│ ● 실시간   │                               │
└──────────┴───────────────────────────────┘
```

### 구조

- 최상위 컨테이너: `display: grid; grid-template-columns: <sidebar-width> 1fr;`
  (사이드바 폭 약 210px). 사이드바는 `position: sticky; top:0; height:100vh`로
  고정하고, 본문만 스크롤한다.
- **사이드바** (`#sidebar`)
  - 상단: 브랜드 — 기존 `.brand`(✦ 마크 + 타이틀 + 서브타이틀) 재사용.
  - 중앙: 세로 메뉴 `#nav` — 버튼 5개(세션/MCP/스킬/메모리/설치)를 세로 스택.
    현재 메뉴는 기존 탭 active 스타일(보라 그라데이션 + 그림자)을 재사용해 강조.
    hover 시 텍스트 밝아짐.
  - 하단(`margin-top:auto`): `실시간 · 5초` 펄스 칩.
- **본문** (`#content`)
  - 상단 바(`#topbar`): `Claude Code v…` 칩을 우측 정렬. 업데이트 대기 시 경고색
    (`chip warn`)은 기존 `initHeader()` 로직 그대로.
  - KPI 바: 기존 `#kpi-bar` 그리드(`repeat(auto-fit, minmax(150px,1fr))`) 유지,
    본문 폭 기준으로 배치.
  - `main`: 기존 패널들(`#panel-*`) 그대로.

### 반응형

- `max-width: 760px`에서 그리드를 1단으로 바꾸고, 사이드바를 상단 가로 스트립으로
  전환한다. 메뉴 버튼은 가로로 wrap, 브랜드는 한 줄로 축소. 최소 구현만 한다.

## 2. 메모리 UI — 유형별 색상 카드 그리드 (1안)

`renderMemory()`(`public/app.js`)를 카드 그리드로 재작성한다. 데이터 형태
(`{ project, files:[{ name, size, description, type }] }`)는 그대로 사용한다.

### 유형 색·아이콘 매핑

| type      | 라벨    | 색          | 아이콘 |
|-----------|--------|-------------|-------|
| user      | 사용자  | 파랑 `#56b6ff` | 👤 |
| feedback  | 피드백  | 노랑 `#ffcc33` | 💬 |
| project   | 프로젝트 | 보라 `#b06cff` | 📌 |
| reference | 참고    | 초록 `#3ddc84` | 🔗 |
| (없음/MEMORY.md) | 색인 | 회색 `#9aa3b8` | 🗂️ |

기존 `MEM_TYPE` 라벨 맵을 유지하고, 색/아이콘 맵(`MEM_STYLE` 등)을 새로 추가한다.
`type`이 없고 파일명이 `MEMORY.md`이면 "색인"으로 처리한다.

### 패널 구성

1. **인트로**: 기존 `.tab-intro` 문구 유지.
2. **유형별 개수 요약**: 전체 메모리를 유형별로 집계해 칩으로 표시
   (예: `사용자 3 · 피드백 2 · 프로젝트 4 · 참고 1`). 각 칩은 해당 유형 색.
3. **프로젝트별 블록**: `📁 프로젝트명 + 개수` 헤더 + 카드 그리드.

### 카드 1개 구성

- 좌측 세로 색 띠(`::before`, 유형 색).
- 상단행: 유형 아이콘(연한 배경 사각형) + 파일명(굵게) + 우측 유형 라벨 칩.
- 본문: 설명(`description`, 한국어 번역 `memTr` 우선 — 기존 로직 유지).
- 하단: 파일 크기(KB). `size` 바이트를 KB로 포맷(`fmtKB`).

그리드는 `repeat(auto-fill, minmax(240px, 1fr))`로 화면 폭에 맞춰 자동 줄바꿈한다.
세션 카드(`.card`)와 톤(그라데이션 배경, 보더, radius)을 맞춰 일관성을 유지한다.

## 수정 대상 파일

| 파일 | 변경 |
|------|------|
| `public/index.html` | `#sidebar` + `#content` 2단 구조로 마크업 재구성. 탭 버튼을 사이드바 `#nav` 안으로 이동. KPI 바와 칩을 본문 영역으로 이동. |
| `public/styles.css` | 2단 그리드 레이아웃, 사이드바/`#nav`/`#topbar` 스타일, 메모리 요약 칩·카드·유형 색, 반응형(`760px`). 기존 `#tabs` 규칙은 사이드바용으로 대체. |
| `public/app.js` | `renderMemory()` 카드 그리드+요약으로 재작성. `fmtKB` 추가. `MEM_STYLE`(색/아이콘) 추가. `setupTabs()`는 nav 버튼(`.tab` 또는 `.nav-item`) 셀렉터에 맞게 조정 — 동작 로직(active 토글 + lazy render)은 동일. |

## 동작/데이터 흐름 (변경 없음 확인)

- 폴링(`tick()` 5초): `refreshOverview()` → KPI 갱신, `refreshSessions()` → 세션 카드.
- 탭/메뉴 클릭 시 해당 패널 `render*()` lazy 호출. 메모리는 `renderMemory()`만 교체.
- API 응답 스키마 불변 → 서버·테스트 영향 없음.

## 에러 처리

- 기존과 동일하게 fetch 실패 시 빈 목록/기본 텍스트로 폴백. 메모리가 없으면
  "메모리 없음" 표시(기존 문구 유지).
- 알 수 없는 `type`은 색인(회색)으로 폴백.

## 테스트 / 검증

- 자동화 테스트는 서버/소스 대상이라 영향 없음(`npm test` 그대로 통과해야 함).
- 수동 검증: `npm start` 후 브라우저에서
  1. 좌측 메뉴 클릭 시 현재 메뉴 강조 + 패널 전환 확인.
  2. 메모리 탭에서 유형별 색/아이콘/요약/카드 그리드 렌더 확인.
  3. 창 폭을 760px 이하로 줄여 사이드바가 상단 스트립으로 접히는지 확인.
